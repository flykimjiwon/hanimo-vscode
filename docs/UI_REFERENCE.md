## 1. VS Code 패널 제목줄

VS Code 가 **자동으로 그려주는 부분**. 우리가 등록한 명령들이 아이콘으로 표시됨.

| 아이콘 | 라벨 | 동작 |
|:---:|---|---|
| ➕ | 새 채팅 | `hanimoCode.newChat` — 빈 새 세션 시작 |
| 🕐 | 채팅 기록 | `hanimoCode.openHistory` — HISTORY 탭으로 이동 |
| ⚙ | 설정 | `hanimoCode.openSettings` — 설정 탭으로 이동 |
| ─ | 접기 | VS Code 기본 — 사이드바 접기 |
| ✕ | 닫기 | VS Code 기본 — 뷰 닫기 (다시 열려면 액티비티 바 ✦ 클릭) |

---

## 2. 우리 탭 헤더

| 탭 | 동작 |
|---|---|
| **CHAT** | 메인 채팅 화면. 빈 상태에서 추천 카드 + 입력창. |
| **HISTORY** | 채팅 기록 리스트. 클릭하여 이전 대화 이어서. |
| **KNOWLEDGE** | `.hanimo.md` + `.hanimo-knowledge/*.md` 편집. 마크다운 미리보기 토글. |
| **SKILLS** | `.hanimo-skills/*.md` 편집. AI가 필요할 때 자동 참고. |

## 4. CHAT 탭 — 메시지 영역

### 사용자 말풍선
- 파란색 배경 (브랜드 액센트), 우측 정렬

### 어시스턴트 응답
- 마크다운 렌더링 (제목/볼드/리스트/링크/인라인 코드)
- 코드 블록은 별도 박스로 분리 (아래 참고)
- 스트리밍 중에는 끝에 `▍` 깜빡이는 커서

### 도구 호출 (⚙ 박스)
- 클릭하여 펼치기 — 도구 인자 + 결과 표시
- 상태 배지: `완료` (녹색) / `중단` (빨강) / 진행 중에는 스피너

### 코드 블록 박스
| 버튼 | 동작 |
|---|---|
| **복사** | 클립보드로 코드 복사 |
| **Diff** | 적용 전 미리보기 — 현재 파일 ↔ 제안 코드 (좌/우 분할) |
| **적용 → path** | `WorkspaceEdit` 으로 파일에 적용. 적용 직후 자동으로 before/after diff 탭이 열림 |
| 언어 라벨 | 단순 표시 (클릭 안 됨) |

> "적용" 버튼은 코드블록의 **첫 줄에 `// path/to/file.ts` 같은 주석**이 있을 때만 표시됨. AI가 응답에서 이 형식을 따르도록 시스템 프롬프트에 안내됨.

## 5. Composer (입력창)

```
┌─────────────────────────────────────────────────┐
│ [@kb/api.md ✕]  [📷 ✕]                          │ ← 첨부 칩 (있을 때만)
│ ┌───────────────────────────────────────────┐   │
│ │ 입력 텍스트...                             │   │
│ └───────────────────────────────────────────┘   │
│ +  </> [Super ⌄]  ⚙           [↑ 전송]         │
└─────────────────────────────────────────────────┘
```

### 텍스트 입력
- **Enter**: 전송
- **Shift+Enter**: 줄바꿈
- **한/일/중 IME**: Enter는 commit이고 send 아님 (자동 처리)
- **이미지 paste/drop**: 클립보드에 이미지가 있을 때 Cmd+V (또는 드래그) → 첨부 칩 추가

### 좌하단 액션 버튼

| 아이콘 | 라벨 | 동작 |
|:---:|---|---|
| ➕ | 첨부 | 드롭다운 — 지식 파일 / 스킬 목록에서 골라 @-멘션 추가 |
| `</>` | 코드블록 | 입력창에 삼중백틱 코드블록 삽입 (선택 영역 있으면 감쌈) |
| **Super ⌄** | 모드 | 드롭다운 — Super / Deep Agent / Plan 선택 |

### 우하단 전송 버튼
| 상태 | 표시 | 동작 |
|---|:---:|---|
| 대기 | ↑ 화살표 (브랜드 블루) | 클릭 — 메시지 전송 |
| 스트리밍 중 | ⏹ 사각형 (회색) | 클릭 — LLM 응답 중단 |
| 비활성 | 흐림 | API 키 미설정 또는 입력 비어있음 |

## 6. BottomBar (하단)

```
💻 폴더명 ⌄        🛡 권한 ⌄              model · 7,666 tok
```

| 칩 | 동작 |
|---|---|
| **💻 [폴더명] ⌄** | 클릭 — 워크스페이스 폴더를 Finder (macOS) / Explorer (Windows) 에서 열기 |

| 우측 텍스트 | 현재 모델명 + 누적 토큰 (마지막 LLM 응답 기준) |

## 8. KNOWLEDGE 탭

상단 토글:
| 버튼 | 동작 |
|---|---|
| **.hanimo.md (프로젝트)** | 워크스페이스 루트의 단일 파일 |
| **지식 폴더** | `.hanimo-knowledge/*.md` 다중 파일 |

### `.hanimo.md` 모드
| 버튼 | 동작 |
|---|---|
| 미리보기 | 마크다운 렌더 |
| 편집 | textarea 편집 |
| 저장 | 파일에 쓰기 (없으면 생성) |

### 지식 폴더 모드
좌측 (파일 리스트):
| 버튼 | 동작 |
|---|---|
| ➕ 새 파일 | prompt → 새 .md 생성 (템플릿 자동) |
| ↻ | 새로고침 |
| 파일 항목 클릭 | 우측에 로드 |
| 🗑 (호버) | 파일 삭제 (확인 모달) |
| `⏸` 접두 | 비활성 파일 (`_` 로 시작하는 파일) |

우측 (뷰어/에디터):
| 버튼 | 동작 |
|---|---|
| 미리보기 | 마크다운 렌더 |
| 편집 | textarea |
| 저장 | 변경사항 디스크에 쓰기 |

> **자동 컨텍스트 주입**: 활성 .md 파일들이 모든 채팅의 시스템 프롬프트에 자동 prepend.

## 9. SKILLS 탭

KNOWLEDGE 와 동일 레이아웃이지만 **본문은 prepend 안 됨** — 인덱스(이름+설명)만 시스템 프롬프트에 노출. AI가 필요할 때 `file_read` 로 본문 로드.

| 버튼 | 동작 |
|---|---|
| ➕ 새 스킬 | YAML frontmatter (name, description) 템플릿 자동 생성 |
| 파일 항목 hover | 툴팁에 description 표시 |
| ↻ | 새로고침 |

스킬 파일 형식:
```markdown
---
name: code-review
description: PR 코드 리뷰 체크리스트
---

## When to use
- ...

## How to apply
- ...
```

## 10. SETTINGS 화면

### 언어
- 한국어 / English 토글

### API
| 필드 | 설명 |
|---|---|
| Base URL | OpenAI-compatible 엔드포인트. 기본: `https://api.novita.ai/openai` |
| API Key | 비밀번호 입력. 저장된 키는 마스킹 표시 (`sk_M...y2I`) |

### 모델 (콤보박스 — 직접 입력 / 자동완성 둘 다)
| 필드 | 설명 |
|---|---|
| Super | 기본 모드 (Plan 모델 비어있을 때 Plan 도 이걸 사용) |
| Deep Agent | 자율 코딩 (긴 도구 호출 루프) |
| Plan | 계획 모드 — 비우면 Super 사용 |

> Onprem 빌드는 Base URL 이 readonly. 사내 엔드포인트 강제. 

### 사용자 지침 (`~/.hanimo/rules.md`)
- 모든 프로젝트의 모든 채팅에 자동 prepend
- 예: "한국어로 응답", "TypeScript 우선", "주석 최소화"

### 인덱싱
- "지금 다시 인덱싱" — `.hanimo-cache/symbols.json` 강제 재빌드 (정규식 기반, ~1초)

## 11. PERMISSIONS 화면 (BottomBar `🛡 권한` 클릭)

### 프로필 (한 번 클릭으로 전체 적용)
| 프로필 | 효과 |
|---|---|
| **🔓 Open** | 모든 도구 허용 (기본값) |
| **🛡 Safe** | 위험 도구(쓰기/실행)는 Ask 모드 |
| **👁 Read-only** | 모든 쓰기/실행 도구 차단 |

### 도구별 (15개)
각 도구마다 `허용` / `확인` / `차단` 버튼:
- **허용**: 즉시 실행
- **확인**: 모달 팝업 → 사용자 응답 대기 (5분 타임아웃)
- **차단**: 정책에 의해 거부, 결과로 "[permission denied]" 반환

### 셸 명령 차단 패턴 (정규식)
- `shell_exec` 호출 시 명령어가 매치되면 거부
- 기본값:
  - `\brm\s+-rf\s+/`, `\brm\s+-rf\s+\*`
  - `\bsudo\s+rm\b`
  - `\bgit\s+push\s+.*--force\b`
  - `\bcurl.*\|\s*sh\b`, `\bcurl.*\|\s*bash\b`
  - fork bomb, mkfs, dd to /dev/sd*, chmod 777 /

저장 위치: `~/.hanimo/permissions.yaml` (TUI 와 공유)

## 12. 도구 (15개) — AI가 호출

| 분류 | 도구 | 기본 권한 |
|---|---|:---:|
| 읽기 | `file_read`, `list_files`, `glob_search`, `grep_search`, `hashline_read`, `diagnostics`, `knowledge_search` | 허용 |

| 쓰기 | `file_write`, `file_edit`, `hashline_edit`, `apply_patch` | 허용 |

## 16. 컨텍스트 주입 우선순위 (시스템 프롬프트 구성)

채팅 시 시스템 프롬프트는 다음 순서로 합쳐짐:

```
1. 모드별 기본 시스템 프롬프트 (Super/Deep/Plan)
2. ~/.hanimo/rules.md                ← 사용자 지침 (모든 프로젝트)
3. <workspace>/.hanimo.md         ← 프로젝트 지식 (단일)
4. <workspace>/.hanimo-knowledge/*.md ← 지식 폴더 (활성 파일만, _ 접두 제외)
5. <workspace>/.hanimo-skills/*.md   ← 스킬 인덱스 (이름 + 설명만, 본문 X)
```

총합 100KB 초과 시 잘림.

@-멘션은 **이번 메시지에만** 추가됨 (한 번만 적용).

## 17. 데이터 영속화 위치

| 데이터 | 위치 | 공유 |
|---|---|---|
| 채팅 기록 | VS Code globalState | PC 전체 |

| UI 언어 선택 | localStorage | WebView 한정 |
